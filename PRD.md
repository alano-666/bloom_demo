## 1. 产品概述

### 1.1 产品名称

**Bloom**（中文名：绽放）

### 1.2 产品一句话

> **一个能够长期理解用户、陪伴用户成长，并主动帮助用户完成每一天成长目标的 AI Growth Agent。**

### 1.3 产品定位

从 **Task-first AI** 走向 **Human-first AI**。

当前主流 AI 产品均围绕"任务"设计（ChatGPT 回答问题、Perplexity 搜索信息、Manus 完成任务、Notion AI 辅助工作），而 Bloom 是第一个围绕"人"进行**长期陪伴式成长服务**的 AI 产品。

### 1.4 核心价值体系

```
Remember Me（记住我）
    ↓
Guide Me（引导每一天）
    ↓
Reflect Me（帮助我成长）
```

三个层次形成成长闭环：

**记住** → **引导** → **反思** → **进步**

## 2. 产品定位与愿景

### 2.1 Problem（产品问题）

当 AI 越来越聪明，人却越来越难成长。

过去几年，AI 极大降低了信息获取和内容创作的门槛，但人的成长效率并没有同步提升。年轻人在学习、工作、健康、生活中每天都需要做大量决策，但缺乏一个能够长期理解他们、持续提供成长指导的工具。

**核心洞察：**
> AI 越来越了解世界，却没有越来越了解用户。

### 2.2 Vision（产品愿景）

打造一个能够长期理解用户、陪伴用户成长，并主动帮助用户完成每一天成长目标的 AI Growth Agent。

无论用户处于：学生 → 求职 → 工作 → 创业 → 家庭 → 退休，AI 都能够持续积累对用户的理解，帮助用户形成长期成长路径，而不是一次次重新开始。

**关键词：长期理解。**

### 2.3 Why Now（为什么是现在）

三个条件第一次同时成熟：

1. **LLM 开始拥有长期记忆能力**
2. **Agent 开始拥有主动执行能力**
3. **越来越多人开始把 AI 当作日常伙伴**

Growth Agent 开始成为可能。

## 3. 目标用户与用户画像

### 3.1 第一阶段目标用户

| 维度 | 描述 |
|------|------|
| 年龄 | 22~30 岁 |
| 特征 | 初入职场、面临多重成长任务 |
| 性别（首期） | 女性占比更高（种子用户策略） |
| 行为 | 高频使用 AI、有成长焦虑 |
| 场景 | 职业发展、健康管理、生活独立 |

### 3.2 用户画像

#### 用户 A：小鹿（24 岁，产品助理）

- **背景**：本科毕业 2 年，在一家互联网公司做产品助理
- **痛点**：每天被琐碎工作填满，感觉没有成长；知道要学习但不知道学什么；投了很多简历没回复，开始自我怀疑
- **需求**：有人帮她把"大目标"拆解成"每天可执行的小步骤"，并在迷茫时给予情绪支持
- **使用场景**：早上通勤时看 AI 建议今天做什么；下班前记录今天做了啥；睡前看 AI 的复盘

#### 用户 B：小羽（27 岁，运营专员）

- **背景**：工作 5 年进入瓶颈期，想转行但不知道方向
- **痛点**：每天很忙但没有积累感；想学习新技能但坚持不下来；经常忘记自己一周前设定的目标
- **需求**：需要有人持续追踪她的进度并及时提醒，想要看到自己"真的在进步"的证据
- **使用场景**：周末做周计划；每天记录小进展；每周看成长报告获得成就感

## 4. 用户洞察与 JTBD

### 4.1 核心用户洞察

真正影响成长的，不是知识不足，而是**成长缺乏连续性**。

用户每天的状态：
```
今天努力学习 → 明天忘记 → 后天重新开始
```

AI 需要提供的三大能力：

| 能力 | 说明 |
|------|------|
| 长期记忆 (Long-term Memory) | 记住用户的目标、习惯、健康、情绪模式 |
| 主动规划 (Proactive Planning) | 每天主动告诉用户"今天应该做什么、为什么、什么时候" |
| 成长反馈 (Growth Reflection) | 每天总结成长了什么；每月对比能力变化 |

### 4.2 Jobs To Be Done

| 类型 | 描述 |
|------|------|
| **Functional Job** | 帮助用户建立长期成长系统：长期记忆重要事件、每日成长规划、决策支持、健康提醒、工作安排、长期目标追踪 |
| **Emotional Job** | 帮助用户在焦虑、迷茫、低谷、犹豫时，获得稳定、持续、可信赖的支持 |
| **Social Job** | 帮助用户持续履行不同人生角色（学生、员工、伴侣、家庭成员等）的责任，提升自我效能感 |

## 5. 核心功能需求

### 5.1 功能总览

MVP 只有一个核心功能：**Daily Growth Session（每日成长会话）**

> 产品只做一件事：每天帮助用户完成"今天最值得完成的成长目标"。

### 5.2 功能详情

#### F1：初始建档（Onboarding）

| 项目 | 描述 |
|------|------|
| **目标** | 1 分钟内完成初始建档，让 AI 初步了解用户 |
| **用户输入** | ① 年龄 ② 当前阶段（学生/求职/职场/创业） ③ 最想成长的方向（职业/健康/学习/生活） ④ 一个长期目标 ⑤ 最近最困扰的一件事 |
| **AI 输出** | 确认已记住用户信息，并给出第一天的成长方向建议 |
| **成功标准** | 用户完成率 > 80%；平均完成时间 < 90 秒 |

示例 AI 回复：
> "我已经记住了你的长期目标：进入字节产品岗。接下来我会帮助你把这个目标拆解成每天可以完成的小步骤。"

#### F2：成长事件录入（Growth Event Entry）

| 项目 | 描述 |
|------|------|
| **目标** | 让用户轻松记录每天的成长事件 |
| **输入方式** | ① 文本输入（主要） ② 语音输入 ③ 截图/图片上传 |
| **AI 分析维度** | 记忆更新（关联到用户目标）、情绪识别、能力评估、行动建议 |

示例：
> 用户："今天看了 2 个产品经理 JD，但感觉要求都好高。"
> AI 回复体现：记忆（"你之前说想进字节"）、情绪识别（"看起来你有点焦虑"）、能力评估（"其实你已经在积累产品技能了"）、行动建议（"明天可以拆解其中 1 个 JD 的能力要求，对比自己的差距"）

#### F3：主动提醒（Proactive Reminder）

| 场景 | AI 行为 | 触发条件 |
|------|---------|---------|
| 轻度遗忘 | 轻提醒 | 24h 未记录 |
| 持续遗忘 | 询问是否遇到困难 | 连续 3 天未记录 |
| 生理期 | 调整成长目标强度 | 经期临近 |
| 重要事件 | 推送复习/准备清单 | 面试前一天 |
| 目标停滞 | 提醒重新规划 | 长期目标停滞 |

示例文案：
> "你已经两天没有记录成长事件了。上次你提到正在准备产品经理面试，需要我帮你把明天的任务重新拆得更轻一点吗？"

#### F4：成长报告（Growth Report）

| 项目 | 描述 |
|------|------|
| **频次** | 每周自动生成（MVP 重点展示） |
| **报告内容** | 本周成长摘要、关键事件回顾、能力变化趋势、下周展望 |
| **设计目标** | ① 用户获得成就感 ② 形成社交传播（可分享） |
| **展示形式** | 信息卡片 + 图文报告 |

#### F5：决策辅助（Decision Assistant — MVP 亮点功能）

| 项目 | 描述 |
|------|------|
| **目标** | 帮助用户在重要决策时获得结构化的分析框架 |
| **示例场景** | "应该接这个 Offer 吗？""要不要搬去另一个城市？" |
| **AI 行为** | 结合用户长期目标、当前状态、历史模式，提供多维度分析建议 |

## 6. AI 工作流

### 6.1 核心工作流

```
用户注册 / 首次使用
    │
    ▼
Onboarding（1 分钟建档）
    │
    ▼
每日成长会话 ←──────────────┐
    │                        │
    ├─ 查看 AI 今日建议       │
    ├─ 记录成长事件           │
    ├─ 接收 AI 反馈与指导     │
    └─ 提出决策问题           │
    │                        │
    ▼                        │
AI 主动提醒（遗忘触发）──────┘
    │
    ▼
每周成长报告（自动生成）
```

### 6.2 AI 能力需求

| 能力 | 优先级 | 说明 |
|------|--------|------|
| 长期记忆 | P0 | 记住用户目标、习惯、健康状态、情绪模式、关键事件 |
| 成长规划 | P0 | 根据长期目标分解到每天可执行的小步骤 |
| 情绪识别 | P1 | 从用户语言中识别焦虑、低落、兴奋等情绪状态 |
| 主动提醒 | P0 | 根据遗忘程度和场景触发不同强度的提醒 |
| 趋势分析 | P1 | 对比用户周/月的能力变化、目标进展、压力水平 |
| 决策框架 | P1 | 为用户提供结构化的决策分析 |

├── 首页 / 每日会话（Daily Session） ← 核心页面
│   ├── AI 今日建议（顶部）
│   ├── 对话流（成长事件记录 + AI 反馈）

```
[欢迎页] → [Onboarding 建档]
                │
                ▼
          [每日对话] ← ─ ─ ─ ─ ─ ─ ┐
                │                   │
        ┌───────┼───────┐           │
        │       │       │           │
        ▼       ▼       ▼           │
    [记录事件] [决策]  [查看]       │
        │       │       │           │
        └───────┼───────┘           │
                │                   │
                ▼                   │
          [AI 反馈] ─── 第二天 ──────┘
                │
                ▼ (每周自动)
          [成长报告]
                │
                ├─ [分享卡片]
                └─ [返回每日会话]
```

### 7.3 核心路径（Happy Path）

```
Day 1:  注册 → Onboarding → 查看 AI 今日建议 → 记录第一件成长事件
Day 2:  打开 App → AI 问候并回顾昨日 → 查看今日建议 → 记录新事件
Day 3-6: 持续每日记录（偶尔收到 AI 提醒）→ AI 越来越了解用户
Week 1: 收到首份周成长报告 → 分享到社交圈 → 获得正反馈
```

## 8. 交互设计需求

### 8.1 核心交互一：成长事件记录

```
用户发送文本 → AI 自动分析 →
  ├─ 记忆更新："我帮你记住了这个进展"
  ├─ 情绪识别："看起来你有点焦虑"
  ├─ 能力评估："这个技能已经在积累了"
  └─ 行动建议："明天建议你做 X"
```

**体现能力：记忆更新、情绪识别、能力评估、行动建议**

### 8.2 核心交互二：决策辅助（亮点功能）

```
用户提出决策困惑 → AI 结合长期记忆 →
  ├─ 当前状态分析
  ├─ 选项利弊对比
  ├─ 与长期目标的一致性评估
  └─ 建议下一步行动
```

### 8.3 交互设计原则

| 原则 | 说明 |
|------|------|
| 极简启动 | 用户打开 app 只需做一件事：记录/查看今天成长 |
| 对话式交互 | 以聊天为主界面，降低使用门槛 |
| 主动而非被动 | AI 在用户遗忘时主动提醒，而不是等用户来问 |
| 肯定优先 | 反馈始终从肯定开始（"你已经做了 X，很不错"），再给建议 |
| 渐进理解 | 不要求用户一次性填完所有信息，在对话中逐步了解 |

## 9. UI 设计规范

### 9.1 设计风格

| 维度 | 规范 |
|------|------|
| 风格 | 极简、温暖、人性化 |
| 主色调 | 柔和暖色系（浅粉/珊瑚/暖白），传达温暖陪伴感 |
| 字体 | 无衬线字体，大号标题，清晰可读 |
| 布局 | 卡片式设计，圆角，充足留白 |
| 动效 | 轻柔过渡，不喧宾夺主 |

### 9.2 关键页面设计指引

#### Onboarding 页
- 单步骤输入，每步只问一个问题
- 大号 Emoji 图标引导
- 进度指示器显示完成进度
- 底部"下一步"按钮，最后一步变为"开始成长"

#### 每日会话页（核心）
- 顶部：日期 + 简洁问候
- 中部：对话流，AI 气泡在左（带柔和背景色），用户气泡在右
- 底部：输入框 + 快捷操作（语音/图片）
- AI 消息支持 Markdown 格式（列表、加粗等）

#### 成长报告页
- 顶部：周数 + 评语（如"你这一周进步很大"）
- 统计卡片：记录天数、事件数、目标完成率
- 关键事件时间线（纵向）
- 能力雷达图（按方向：职业/健康/学习/生活）
- 底部：分享按钮

### 9.3 Logo 与品牌

- Logo 设计：Bloom = 绽放，使用花朵/萌芽意象
- 品牌色：#FF8C8C（珊瑚红）、#FFD9D9（浅粉）、#2D2D2D（深灰文字）
- App 名称展示统一使用 "Bloom"

## 10. 非功能需求

### 10.1 性能要求

| 指标 | 目标 |
|------|------|
| 页面加载时间 | < 2s（首屏） |
| AI 响应时间 | < 3s |
| 支持同时在线用户 | MVP 阶段 > 100 |
| 可用性 | 99.5% |

### 10.2 技术架构建议

```
前端（React + TypeScript + Tailwind CSS）
    │
    ├── 移动端优先的响应式设计
    ├── PWA 支持（可添加到主屏幕）
    └── 离线能力（核心对话缓存）
    │
后端（Node.js + Express）
    │
    ├── RESTful API
    ├── LLM 集成层（支持 OpenAI / Anthropic）
    └── 长期记忆存储（PostgreSQL + 向量数据库）
```

### 10.3 安全与隐私

- 用户数据加密存储
- 支持账号注销与数据删除
- 成长报告分享前需用户确认
- 语音数据不上传服务器（可本地处理）

## 11. 成功指标

### 11.1 MVP 阶段核心指标

| 指标 | 目标值 | 说明 |
|------|--------|------|
| Onboarding 完成率 | > 80% | 注册用户完成建档的比例 |
| Day-7 留存率 | > 40% | 第 7 天仍活跃的用户比例 |
| 日均记录数 | > 1.5 条/天 | 活跃用户每天记录的成长事件数 |
| 周报分享率 | > 15% | 收到周报的用户中分享的比例 |
| NPS | > 40 | 用户推荐度 |

### 11.2 体验指标

- **"它记住我了"时刻**：用户首次感受到 AI 记住自己之前说过的话
- **"它懂我的问题"时刻**：AI 对用户困扰给出了超出预期的理解
- **"我真的很在进步"时刻**：用户看到成长报告时感到成就感

## 12. MVP 范围与砍掉的功能

### 12.1 MVP 核心范围

| 功能 | 优先级 | 说明 |
|------|--------|------|
| Onboarding 建档 | P0 | 必做 |
| 每日成长会话（文本） | P0 | 核心体验 |
| AI 长期记忆 | P0 | 需要在对话中体现 |
| 成长周报 | P0 | Demo 展示重点 |
| 主动提醒 | P1 | 基础版本即可 |
| 语音输入 | P1 | MVP 后期 |
| 截图上传 | P2 | MVP 后期 |
| 决策辅助 | P1 | Demo 亮点 |

### 12.2 明确砍掉的功能

| 功能 | 原因 |
|------|------|
| 社区/社交 | 会分散用户注意力，降低成长完成率 |
| 信息流/发现 | 用户会从"成长"变成"消费内容" |
| 商城/商业化 | MVP 阶段不验证商业化 |
| 多 AI 角色 | 增加认知负担，偏离核心体验 |
| 复杂任务自动执行 | 技术复杂度高，无法在 Demo 阶段稳定实现 |

**MVP 的目标不是"什么都能做"，而是验证一个核心假设：**

> 用户是否愿意每天回来和 AI 一起完成成长。

## 13. Roadmap

### Phase 1：MVP（v1.0 — 当前阶段）

**目标：验证核心假设**
- Onboarding 建档
- 每日成长会话（文本）
- AI 长期记忆 + 基本反馈
- 周成长报告
